## 1. Claude Codeとは？ — ターミナルで動くAIエンジニア

Claude Codeは、Anthropic社が提供するCLIツールです。

ポイントは「チャットではない」ということ。ターミナルの中で直接コードを読み、書き、実行できるAIエージェントです。

普通のAIチャット（ChatGPT、Geminiなど）との最大の違いは、**プロジェクト全体のコンテキストを理解した上でコードを書ける**こと。ファイルを1つずつコピペする必要がありません。

```bash
# インストール
npm install -g @anthropic-ai/claude-code

# プロジェクトのルートで起動
cd my-project
claude
```

これだけで、プロジェクトの構造を理解し、既存のコードパターンに合わせた提案をしてくれます。

## 2. ChatGPTとの決定的な違い

多くのエンジニアが「ChatGPTでコード生成してるから十分」と思っています。私もそうでした。

しかし、実際に使い比べると、根本的に異なるツールだとわかります。

| 比較項目 | ChatGPT / Gemini | Claude Code |
|---------|-----------------|-------------|
| 動作環境 | ブラウザ | ターミナル（ローカル） |
| コンテキスト | コピペしたコードのみ | プロジェクト全体を自動読み込み |
| 出力 | チャット欄にコード表示 | 直接ファイルを編集 |
| テスト | 自分で実行 | 書いて実行まで自動 |
| Git操作 | 不可 | コミット・PR作成まで可能 |
| 依存関係 | 把握不可 | package.json等を自動参照 |

一番の違いは**「コンテキスト」**です。

ChatGPTに「このバグを直して」と言うには、関連ファイルをすべてコピペする必要があります。5ファイルにまたがるバグなら、5回コピペです。

Claude Codeなら「このエラーを修正して」と言うだけ。関連ファイルを自動で探し、依存関係を追い、修正を提案します。

## 3. 開発速度が10倍になった5つの使い方

### 使い方①：バグ修正を丸投げする

エラーログをそのまま貼り付けるだけです。

```
このエラーを修正して:
TypeError: Cannot read properties of undefined (reading 'map')
```

Claude Codeは以下を自動で行います：
1. エラーの発生箇所を特定
2. 関連するファイルを読み込み
3. 原因を分析
4. 修正コードを提案 → 承認すれば直接適用

以前は30分〜1時間かかっていたバグ修正が、5分で終わります。

### 使い方②：テストを自動生成する

テストを書くのが面倒で後回しにしていませんか？

```
この関数のユニットテストを書いて。エッジケースも含めて。
```

Claude Codeはコードの実装を読んだ上で、意味のあるテストケースを生成します。「引数がnullの場合」「配列が空の場合」など、見落としがちなエッジケースも含めてくれます。

実感として、テストカバレッジが30%から80%に上がりました。テスト作成にかかる時間は1/5以下です。

### 使い方③：リファクタリングを安全に実行する

「このファイル、600行もあるけど分割したい...でも依存関係が怖い」

こういう場面こそClaude Codeの真価が発揮されます。

```
このファイルを責務ごとに分割して。既存のインポートも全て更新して。
```

プロジェクト全体のインポート関係を把握しているので、分割後に壊れることがほぼありません。600行のファイルを3つのモジュールに分割する作業が、レビュー込みで15分で完了しました。

### 使い方④：新機能の実装を対話的に進める

Claude Codeの最も強力な使い方は「対話的な実装」です。

```
ユーザー認証機能を追加して。JWT + リフレッシュトークン方式で。
既存のデータベーススキーマに合わせて。
```

すると、Claude Codeは：
1. 既存のDB設計を確認
2. 認証に必要なモデルを追加
3. ルーターとミドルウェアを実装
4. テストも一緒に作成

途中で「パスワードのハッシュ化はbcryptとargon2どちらを使いますか？」のように確認してきます。丸投げではなく、重要な判断は人間に委ねるのが良いところです。

### 使い方⑤：コミットメッセージとPR作成

地味ですが、毎日の積み重ねで最も時間を節約できるのがこれです。

```
変更内容をコミットして。PRも作成して。
```

diffを読んで、適切なコミットメッセージを生成し、PRの説明文まで書いてくれます。

## 4. 今日から使えるコマンド集

Claude Codeを入れたら、まず試してほしいコマンドを紹介します。

```bash
# プロジェクトの全体像を把握
claude "このプロジェクトの構造を説明して"

# 特定ファイルの解説
claude "src/auth/middleware.ts を解説して"

# バグ修正
claude "npm testで失敗するテストを修正して"

# リファクタリング
claude "この関数を読みやすくリファクタして"

# ドキュメント生成
claude "このAPIのドキュメントを生成して"
```

**プロのTip**: `CLAUDE.md` というファイルをプロジェクトルートに置くと、プロジェクト固有のルール（コーディング規約、テストの書き方など）をClaude Codeに教えられます。これだけでアウトプットの品質が格段に上がります。

## 5. やってはいけないアンチパターン

### ❌ アンチパターン①：全部丸投げ

「アプリを作って」のような曖昧な指示は効果的ではありません。

Claude Codeは具体的な指示ほど正確に動きます。「ユーザー登録のAPIエンドポイントを、既存のusersテーブルを使って実装して」のように具体的に伝えてください。

### ❌ アンチパターン②：レビューせずにマージ

Claude Codeが書いたコードも、必ず人間がレビューしてください。

特にセキュリティに関わる部分（認証、入力バリデーション、SQL構築）は、AI任せにせず目を通すべきです。

### ❌ アンチパターン③：CLAUDE.mdを設定しない

プロジェクト固有のルールを教えないと、Claude Codeは一般的なパターンで書きます。`CLAUDE.md`でコーディング規約、使用するライブラリ、テスト方針を明記すると、チーム開発のコードレビューに通るレベルのコードを出力してくれます。

## まとめ

- Claude Codeはターミナルで動くAIエンジニア。プロジェクト全体を理解した上でコードを書ける

- ただし、丸投げ・レビュー省略・設定不足は逆効果
- `CLAUDE.md`の設定が品質を左右する最重要ポイント
